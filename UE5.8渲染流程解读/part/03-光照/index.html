2. 核心概念

2.1 延迟光照 vs 前向光照

UE5.8 默认使用 延迟渲染（Deferred Shading） 管线。在延迟渲染中，Base Pass 将几何体属性（法线、粗糙度、材质 ID 等）写入 G-Buffer，光照计算推迟到 RenderLights 阶段统一进行。每种光源类型至少写入一个全屏或 Tile 限定区域的光照贡献，最终累加到场景颜色。

前向渲染（Forward Shading）仅作为可选路径用于特定平台或启用 r.ForwardShading 时，其光照在 Base Pass 中逐像素计算，不需要 G-Buffer。

2.2 Clustered Deferred Shading — 3D 聚簇网格原理

Clustered Deferred Shading 是 UE5.8 延迟光照计算的核心技术。其核心思想是将相机视锥空间沿 X（屏幕宽）、Y（屏幕高）、Z（深度）三个维度剖分为规则的 3D Tile 网格，每个 Tile（称为一个 Cluster）维护一份对该 Tile 有贡献的光源列表。

相对于传统的 Tiled Deferred（仅 2D 屏幕划分），Clustered 方法在深度维度也做剖分，有效解决了大深度范围内光源过多导致的 culling 效率低下问题。

3. 光源收集与排序

3.1 异步任务启动

GatherAndSortLights 是光源准备流程的总入口。它通过 LaunchSceneRenderTask 作为异步任务启动，与后续的深度 PrePass、Nanite Visibility 等阶段并行执行，仅在需要光照数据时（PrepareForwardLightData）才调用 Wait() 同步。

// DeferredShadingRenderer.cpp L2354-L2368 auto* SortedLightSet = GraphBuilder.AllocObject<FSortedLightSetSceneInfo>(); const bool bShadowedLightsInClustered = ShouldUseClusteredDeferredShading(...) && IsVSMOnePassProjectionEnabled(...) && VirtualShadowMapArray.IsEnabled(); auto GatherAndSortLightsTask = LaunchSceneRenderTask<FSortedLightSetSceneInfo*>( [this, SortedLightSet, bShadowedLightsInClustered] { GatherAndSortLights(*SortedLightSet, bShadowedLightsInClustered); return SortedLightSet; }, IssuedTasksCompletionEvents);

异步任务的依赖链：GetGatherAndSortLightsPrerequisiteTask（等待 Dynamic Shadows 初始化完成）+ UpdateLightFunctionAtlasTask（等待光照函数图集更新完成）。

3.2 GatherAndSortLights 内部逻辑

定义于 LightRendering.cpp L1409，核心流程如下：

GatherAndSortLights(OutSortedLights, bShadowedLightsInClustered)
1. GatherSimpleLights(ViewFamily, Views, OutSimpleLights) — 收集场景中的简单光源（粒子光）
2. 遍历 Scene->Lights，对每个 FLightSceneInfoCompact 检查 ShouldRenderLight
3. 为每个可见光源创建 FSortedLightSceneInfo，填充 SortKey 字段： LightType / bShadowed / bLightFunction / bIESProfile / bUsesLightingChannels / bClusteredDeferredNotSupported / bHandledByMegaLights
4. 判断 bClusteredDeferredSupported — 无阴影/无光照函数/非方向光/无接触阴影/非 MegaLights → 可批处理
5. 调用 MegaLights::GetMegaLightsMode() 判断是否由 MegaLights 处理 → 设置 bHandledByMegaLights
6. 添加 SimpleLights 到 SortedLights 数组（SimpleLightIndex）
7. 对 SortedLights 按 SortKey.Packed 排序 → 自然形成 [Simple | Clustered | Unbatched | MegaLights] 四段
8. 二分查找确定 SimpleLightsEnd / ClusteredSupportedEnd / UnbatchedLightStart / MegaLightsLightStart 边界

3.3 三桶分类结果

Batched（批处理光源）

SimpleLights[0] ~ UnbatchedLightStart

包含 Simple Lights 和满足聚类条件的光源（无阴影、无光照函数、非方向光）。通过 Clustered Deferred Shading 单 Pass 批量计算，或者回退到传统逐光源渲染。

Unbatched（非批处理光源）

UnbatchedLightStart ~ MegaLightsLightStart

包含带阴影、光照函数、IES 配置文件等特殊光源。每个光源独立渲染一个全屏或限定区域的光照 Pass。方向光（Directional Light）始终在此区间。

MegaLights（随机采样光源）

MegaLightsLightStart ~ SortedLights.Num()

UE5.8 新增的大量动态可投影光源系统。由 MegaLights::GetMegaLightsMode() 判定，使用随机采样策略高效处理大量阴影投射光源。

4.1 三阶段流水线

4.2 LightGridInjection 核心机制

LightGridInjection.cpp 中的 ComputeLightGrid 函数是 Clustered Deferred 的 CPU 端核心。它使用 Froxel 网格工具（FroxelGridUtils.h）创建每个视图的 3D 体素化光照网格：

  • CullLightsToGrid：对每个 Froxel Cell，测试所有 Batched 聚类光源的包围球/包围盒，剔除不可见光源
  • InjectLightsToGrid：将剔除后保留的光源索引写入 GPU 可访问的 Structured Buffer 和查找表
  • Light Grid 纹理：每个 Cell 存储 "起始偏移 + 光源数量"，Shader 端通过 GetLightGridCell() 查询
  • CSV 统计：CSV_CUSTOM_STAT(LightCount, Batched, ...) 和 CSV_CUSTOM_STAT(LightCount, Unbatched, ...) 输出调试指标

4.3 光照函数图集（LightFunctionAtlas）

光照函数（Light Function）是一种材质驱动的光源遮罩/衰减自定义机制。传统方式需要为每个带光照函数的光源单独渲染，无法批处理。UE5.8 的光照函数图集机制（FLightFunctionAtlas）将多个光照函数材质渲染到一张 2D 图集纹理中，使得支持图集的光源也能被 Clustered Deferred 批处理。

5. MegaLights

5.1 系统概述

MegaLights 是 UE5.8 引入的实验性大量动态可投影光源系统。它使用 随机光源采样（Stochastic Light Sampling） 策略，每像素仅采样少量光源而非遍历所有可见光源，从而支持场景中数十甚至数百个投射阴影的动态光源，同时保持可预测的低 GPU 成本。

关键限制：MegaLights 需要 Hardware Ray Tracing 支持，且不适用于 Directional Light（方向光）。启用后通过 Post Process Volume 或光源级别（bAllowMegaLights）逐光源控制。

5.2 三种运行模式

由 MegaLights::GetMegaLightsMode() 为每个光源判定运行模式：

模式枚举值说明
DisabledEMegaLightsMode::Disabled不走 MegaLights，走传统光源路径
EnabledRTEMegaLightsMode::EnabledRT使用 Hardware Ray Tracing 计算阴影
EnabledVSMEMegaLightsMode::EnabledVSM使用 Virtual Shadow Maps 计算阴影（回退路径）

5.3 随机光源采样算法

MegaLights 的核心采样策略（MegaLightsSampling.cpp）包含以下关键步骤：

  1. Tile Classification（Tile 分类） — MegaLights::SetupTileClassifyParameters()：将屏幕划分为 Tile，对每个 Tile 评估哪些光源可能对像素产生有意义的贡献。使用历史帧信息（MegaLightsHistoryDistanceThreshold）引导分类。
  2. Sample Generation（采样生成） — GenerateMegaLightsSamples()：在每个像素使用 Blue Noise 序列随机选择 N 个光源（r.MegaLights.NumSamplesPerPixel，默认 4）。通过重要性采样优先选择亮度高、距离近的光源。
  3. Shadow Evaluation（阴影评估） — MegaLightsRayTracing.cpp：对每个采样光源发射光线追踪射线评估可见性（HWRT 路径）或查询 VSM 页面（VSM 路径）。支持 Global SDF 回退（MegaLights::IsUsingGlobalSDF()）。
  4. Lighting Accumulation（光照累积） — Shader 中对每个可见光源评估 BRDF，加权累加漫反射和高光贡献。

5.4 HWRT / SDF 双路径

HWRT（Hardware Ray Tracing）

MegaLights::UseHardwareRayTracing()

主要路径。使用内联光追（Inline Ray Tracing）直接对 BVH 加速结构发射阴影射线，精度最高。通过 MegaLights::UseInlineHardwareRayTracing() 控制内联模式。

VSM（Virtual Shadow Maps）

EMegaLightsMode::EnabledVSM

回退路径，适用于不支持 HWRT 的平台。使用虚拟阴影贴图的预渲染深度信息查询可见性，精度受 VSM 页面分辨率限制。

Global SDF（全局有符号距离场）

MegaLights::IsUsingGlobalSDF()

辅助路径。使用全局距离场的 SDF 追踪作为 HWRT 的补充，主要用于远距离或非精确阴影查询。通过 MegaLights::IsUsingClosestHZB() 可启用最近 HZB 辅助遮挡。

5.5 时序降噪策略

MegaLights 使用两层时序降噪（MegaLightsDenoising.cpp）：

  • 时间累积：使用 MegaLightsHistoryDistanceThreshold 结合重投影，将当前帧的稀疏采样结果与历史帧累积结果混合。历史引导采样（r.MegaLights.GuideByHistory，默认开启）减少对前一帧被遮挡光源的采样概率，降低高对比度阴影区域的噪声。
  • 空间滤波：对直接光照结果应用边缘保持的空间滤波，利用 G-Buffer 的法线和深度信息控制滤波核范围。半透明光照体积也有独立的空间滤波（MegaLights::IsTranslucencyVolumeSpatialFilterEnabled()）。
  • 降采样优化：支持 r.MegaLights.DownsampleMode（0=全分辨率 / 1=棋盘格 2x1 / 2=半分辨率 2x2），在降采样空间做采样和追踪，再上采样融合回全分辨率。

6. 虚拟阴影贴图（VSM）

6.1 VSM 在光照管线中的定位

虚拟阴影贴图（Virtual Shadow Maps）是 UE5.5+ 默认的阴影系统，用于替代传统的级联阴影贴图（CSM）。VSM 使用虚拟化纹理技术，只分配被屏幕像素实际需要的阴影页面（Page），从而实现极高分辨率的阴影（每 Page VSM_PAGE_SIZE = 128x128 像素）同时控制内存开销。

在光照管线中，VSM 与 MegaLights、Clustered Deferred 紧密集成：BeginMarkVirtualShadowMapPages 标记需要渲染的阴影页面，RenderShadowDepthMaps 渲染阴影深度，后续的 RenderVirtualShadowMapProjectionMaskBits 生成投影遮罩供光照 Pass 使用。

RenderDeferredReflectionsAndSkyLighting 内部流程
1. 检查 ReflectionsMethod 管道状态（Lumen / SSR / RT Reflections）
2. Lumen 反射：使用 Lumen Radiance Cache + Surface Cache 追踪反射光线
3. SSR（屏幕空间反射）：屏幕空间光线步进（Screen Space Ray Tracing）
4. 天空光照合成：将 Sky Light 的漫反射和高光贡献合成到场景颜色（仅不透明像素）
5. Bent Normal AO 集成：使用 DynamicBentNormalAOTextures 增强反射方向精度
6. 光追天空光：如果启用且非 Lumen GI 模式，执行 RenderRayTracingSkyLight + CompositeRayTracingSkyLight

8.3 反射方法

方法枚举说明
LumenEReflectionsMethod::LumenLumen 反射追踪，支持 HWRT 和 SDF 双路径
SSREReflectionsMethod::SSR屏幕空间反射
RT Reflections--完整光线追踪反射（可选）

9.1 函数签名

9.2 体素标记机制

半透明光照体积（Translucency Lighting Volume）是 UE5 用于加速半透明物体光照计算的 3D 体素化结构。它在相机周围创建一张 3D 网格，每个体素存储该位置的环境光照和方向光照信息。半透明渲染阶段从此体积采样，避免对每个半透明像素执行昂贵的光照计算。

半透明光照体积处理流程
1. MarkUsedTranslucencyVolumeVoxels() — 标记哪些体素需要处理
  ↳ 使用 MegaLights::UseTranslucencyVolumeMarkTexture() 判断是否由 MegaLights 驱动标记
2. GatherTranslucencyVolumeMarkedVoxels(GraphBuilder, SortedLightSet) — 收集标记的体素
  ↳ 如果 bGatheredTranslucencyVolumeMarkedVoxels 为 false，在 Phase 6 延迟光照前补充收集
3. RenderTranslucencyLightingVolume(GraphBuilder, TranslucencyLightingVolumeTextures, SortedLightSet) — 渲染体积光照
  ↳ 对每个标记的体素，评估 SortedLightSet 中相应光源的贡献
  ↳ 输出 TranslucencyAmbient[TVC_MAX] 和 TranslucencyDirectional[TVC_MAX] 纹理
MegaLights 与半透明体积：MegaLights::UseTranslucencyVolume() 控制是否使用 MegaLights 处理半透明光照体积。如果启用，FMegaLightsVolume 输出 TranslucencyAmbient 和 TranslucencyDirectional 纹理阵列（TVC_MAX 级联），支持空间滤波（IsTranslucencyVolumeSpatialFilterEnabled()）和时间滤波（IsTranslucencyVolumeTemporalFilterEnabled()）。
