1. 在 Render 函数中嵌入自定义 Pass

所有渲染工作都在 FDeferredShadingSceneRenderer::Render() 中通过 FRDGBuilder 注册和执行。添加自定义 Pass 的本质是：在合适的管线段选择一个插入点，向 GraphBuilder 注册一个新的 Pass，声明其输入输出资源，RDG 会自动处理依赖关系和屏障。

插入点 A
BasePass 之后、光照之前
适合写入 GBuffer 扩展数据，或在光照计算前修改 G-Buffer。对应 Render 函数中 RenderBasePass 之后、RenderLights 之前。
插入点 B
光照之后、后处理之前
适合在最终场景颜色上进行自定义后期效果，或额外合成。对应 RenderDeferredReflectionsAndSkyLighting 之后。
插入点 C
后处理链条内
适合添加自定义后处理 Pass。在 AddPostProcessingPasses 中，TSR/TAA 之前或 Tonemap 之前插入。
插入点 D
AsyncCompute 并行
适合与 Graphics Pass 并行执行的独立计算任务。使用 ERDGPassFlags::AsyncCompute，注意资源竞争。
关键原则：永远不要直接调用 RHI 命令（FRHICommandList）。所有 Pass 必须通过 GraphBuilder 注册，由 RDG 统一调度。RDG 会自动管理资源状态转换（RT→SRV→UAV）、屏障插入和 Transient 内存分配。

2. Pass 的本质：Shader + 参数 + 调度

在 UE5 中，一个 Pass 由三个核心元素组成：

元素作用定义位置
Shader 类 封装 GPU Shader 代码（.usf），声明参数结构。派生自 FGlobalShader C++ 头文件中 DECLARE_GLOBAL_SHADER
.cpp 中 IMPLEMENT_GLOBAL_SHADER
参数结构 声明 Pass 所需的所有输入/输出资源。使用 SHADER_PARAMETER_STRUCT 宏体系 嵌入在 Shader 类内部（FParameters 嵌套结构体）
Pass 调度 向 FRDGBuilder 注册 Pass，绑定实际资源，指定 Dispatch/绘制参数 在 Render() 或其子函数中调用 GraphBuilder.AddPass()

Shader 参数宏提供了完整的资源类型映射：

// ---- 纹理资源 ----
SHADER_PARAMETER_RDG_TEXTURE(Texture2D, MyInput)       // RDG 纹理输入 (SRV)
SHADER_PARAMETER_RDG_TEXTURE_SRV(Texture2D, MySRV)     // 显式 SRV
SHADER_PARAMETER_RDG_TEXTURE_UAV(RWTexture2D, MyUAV)   // RDG 纹理输出 (UAV)
SHADER_PARAMETER_RDG_BUFFER(StructuredBuffer, MyBuf)  // RDG Buffer (SRV)
SHADER_PARAMETER_RDG_BUFFER_UAV(RWStructuredBuffer, MyBufUAV) // RDG Buffer (UAV)
// ---- 常量/标量 ----
SHADER_PARAMETER(float, Intensity)
SHADER_PARAMETER(FVector4f, TintColor)
SHADER_PARAMETER_STRUCT_REF(FViewUniformShaderParameters, View)  // View UB 引用
// ---- 渲染目标 (仅光栅化) ----
RENDER_TARGET_BINDING_SLOTS()                        // 光栅化 RT 绑定
所有 RDG 资源（带 RDG_ 前缀的宏）的生命周期由 FRDGBuilder 管理——它们只是"引用"，实际 GPU 内存分配由 RDG 自动延迟到 Pass 执行时。这意味着你可以"创建"任意多的 RDG 纹理而不必担心显存碎片，RDG 会通过别名分析复用 Transient 内存。

3. 光栅化 Pass — 完整模板

光栅化 Pass 需要定义 Vertex Shader + Pixel Shader、绑定 RenderTarget、设置 PSO 状态。

3.1 Shader 定义（.usf）

// MyCustomPass.usf
#include "/Engine/Private/Common.ush"

void MainVS(
    in float4 InPosition : ATTRIBUTE0,
    in float2 InUV : ATTRIBUTE1,
    out float4 OutPosition : SV_POSITION,
    out float2 OutUV : TEXCOORD0)
{
    OutPosition = InPosition;
    OutUV = InUV;
}

Texture2D SceneColorTexture;        // 对应 SHADER_PARAMETER_RDG_TEXTURE
SamplerState SceneColorSampler;
float Intensity;

void MainPS(
    in float4 SVPos : SV_POSITION,
    in float2 UV : TEXCOORD0,
    out float4 OutColor : SV_Target0)
{
    float3 SceneColor = SceneColorTexture.Sample(SceneColorSampler, UV).rgb;
    OutColor = float4(SceneColor * Intensity, 1.0);
}

3.2 Shader 类定义（C++ 头文件）

#pragma once
#include "GlobalShader.h"
#include "RenderGraphResources.h"
#include "ScreenPass.h"

// 像素着色器
class FMyCustomPassPS : public FGlobalShader
{
public:
    DECLARE_GLOBAL_SHADER(FMyCustomPassPS);
    SHADER_USE_PARAMETER_STRUCT(FMyCustomPassPS, FGlobalShader);

    BEGIN_SHADER_PARAMETER_STRUCT(FParameters)
        SHADER_PARAMETER_RDG_TEXTURE(Texture2D, SceneColorTexture)
        SHADER_PARAMETER(SAMPLER, SceneColorSampler)
        SHADER_PARAMETER(float, Intensity)
        RENDER_TARGET_BINDING_SLOTS()
    END_SHADER_PARAMETER_STRUCT()
};

// 顶点着色器 — 通常直接复用引擎提供的全屏三角形 VS
class FMyCustomPassVS : public FGlobalShader
{
public:
    DECLARE_GLOBAL_SHADER(FMyCustomPassVS);
    SHADER_USE_PARAMETER_STRUCT(FMyCustomPassVS, FGlobalShader);

    BEGIN_SHADER_PARAMETER_STRUCT(FParameters)
    END_SHADER_PARAMETER_STRUCT()
};

3.3 Shader 注册（C++ .cpp）

// MyCustomPassRendering.cpp
IMPLEMENT_GLOBAL_SHADER(FMyCustomPassPS, "/Engine/Private/MyCustomPass.usf", "MainPS", SF_Pixel);
IMPLEMENT_GLOBAL_SHADER(FMyCustomPassVS, "/Engine/Private/MyCustomPass.usf", "MainVS", SF_Vertex);
顶点着色器简化：大多数全屏 Pass 不需要自定义 VS——引擎提供了 FPostProcessVS（在 PostProcessShaders.usf 中定义）和 FScreenPassVS。在 AddPass 时直接 TShaderMapRef<FPostProcessVS>(View.ShaderMap) 即可。

3.4 Pass 注册与调度

void AddMyCustomPass(FRDGBuilder& GraphBuilder, const FViewInfo& View, FRDGTextureRef SceneColor)
{
    // 1. 分配参数
    auto* PassParameters = GraphBuilder.AllocParameters<FMyCustomPassPS::FParameters>();
    PassParameters->SceneColorTexture = SceneColor;
    PassParameters->SceneColorSampler = TStaticSamplerState<SF_Bilinear>::GetRHI();
    PassParameters->Intensity = 1.5f;

    // 2. 创建输出纹理
    FRDGTextureDesc OutputDesc = SceneColor->Desc;
    OutputDesc.Flags |= TexCreate_ShaderResource;
    FRDGTextureRef OutputTexture = GraphBuilder.CreateTexture(OutputDesc, TEXT("MyCustomPassOutput"));

    // 3. 绑定渲染目标
    PassParameters->RenderTargets[0] = FRenderTargetBinding(OutputTexture, ERenderTargetLoadAction::EClear);

    // 4. 注册光栅化 Pass
    GraphBuilder.AddPass(
        RDG_EVENT_NAME("MyCustomPass"),
        PassParameters,
        ERDGPassFlags::Raster,
        [&View, PassParameters](FRDGAsyncTask, FRHICommandList& RHICmdList)
        {
            const FIntPoint OutputSize = PassParameters->RenderTargets[0].GetTexture()->Desc.Extent;

            // 设置视口
            RHICmdList.SetViewport(0, 0, 0.0f, OutputSize.X, OutputSize.Y, 1.0f);

            // 构建 PSO
            FGraphicsPipelineStateInitializer GraphicsPSOInit;
            RHICmdList.ApplyCachedRenderTargets(GraphicsPSOInit);

            // 绑定 Shader
            TShaderMapRef<FMyCustomPassVS> VertexShader(View.ShaderMap);
            TShaderMapRef<FMyCustomPassPS> PixelShader(View.ShaderMap);

            GraphicsPSOInit.BoundShaderState.VertexShaderRHI = VertexShader.GetVertexShader();
            GraphicsPSOInit.BoundShaderState.PixelShaderRHI = PixelShader.GetPixelShader();

            // 设置状态（无混合、无深度测试）
            GraphicsPSOInit.BlendState = TStaticBlendState<>::GetRHI();
            GraphicsPSOInit.RasterizerState = TStaticRasterizerState<>::GetRHI();
            GraphicsPSOInit.DepthStencilState = TStaticDepthStencilState<false, CF_Always>::GetRHI();
            GraphicsPSOInit.PrimitiveType = PT_TriangleList;

            SetGraphicsPipelineState(RHICmdList, GraphicsPSOInit, 0);

            // 绑定参数
            SetShaderParameters(RHICmdList, PixelShader, PixelShader.GetPixelShader(), *PassParameters);

            // 绘制全屏三角形（3顶点 = 1三角形覆盖整个视口）
            RHICmdList.DrawPrimitive(0, 1, 1);
        });
}

4. 计算 Pass — 完整模板

4.1 Compute Shader（.usf）

// MyCustomCompute.usf
#include "/Engine/Private/Common.ush"

Texture2D InputTexture;
RWTexture2D<float4> OutputUAV;
float Intensity;
uint2 TextureSize;

[numthreads(8, 8, 1)]
void MainCS(uint3 DispatchThreadId : SV_DispatchThreadID)
{
    if (any(DispatchThreadId.xy >= TextureSize))
        return;

    float3 Color = InputTexture.Load(int3(DispatchThreadId.xy, 0)).rgb;
    OutputUAV[DispatchThreadId.xy] = float4(Color * Intensity, 1.0);
}

4.2 Shader 类定义与注册

// C++ 头文件
class FMyCustomPassCS : public FGlobalShader
{
public:
    DECLARE_GLOBAL_SHADER(FMyCustomPassCS);
    SHADER_USE_PARAMETER_STRUCT(FMyCustomPassCS, FGlobalShader);

    BEGIN_SHADER_PARAMETER_STRUCT(FParameters)
        SHADER_PARAMETER_RDG_TEXTURE(Texture2D, InputTexture)
        SHADER_PARAMETER_RDG_TEXTURE_UAV(RWTexture2D, OutputUAV)
        SHADER_PARAMETER(float, Intensity)
        SHADER_PARAMETER(FVector2f, TextureSize)
    END_SHADER_PARAMETER_STRUCT()
};

// .cpp 注册
IMPLEMENT_GLOBAL_SHADER(FMyCustomPassCS, "/Engine/Private/MyCustomCompute.usf", "MainCS", SF_Compute);

4.3 Pass 调度 — FComputeShaderUtils::AddPass

void AddMyCustomComputePass(FRDGBuilder& GraphBuilder, const FViewInfo& View, FRDGTextureRef SceneColor)
{
    const FIntPoint TextureSize = SceneColor->Desc.Extent;

    // 1. 创建输出 UAV
    FRDGTextureDesc OutputDesc = FRDGTextureDesc::Create2D(
        TextureSize, PF_FloatRGBA, FClearValueBinding::Black,
        TexCreate_ShaderResource | TexCreate_UAV);
    FRDGTextureRef OutputTexture = GraphBuilder.CreateTexture(OutputDesc, TEXT("MyComputeOutput"));

    // 2. 分配并填充参数
    auto* PassParameters = GraphBuilder.AllocParameters<FMyCustomPassCS::FParameters>();
    PassParameters->InputTexture = SceneColor;
    PassParameters->OutputUAV = GraphBuilder.CreateUAV(OutputTexture);
    PassParameters->Intensity = 1.5f;
    PassParameters->TextureSize = FVector2f(TextureSize.X, TextureSize.Y);

    // 3. 获取 Shader
    TShaderMapRef<FMyCustomPassCS> ComputeShader(View.ShaderMap);

    // 4. 计算线程组数量
    FIntVector GroupCount = FComputeShaderUtils::GetGroupCount(
        FIntVector(TextureSize.X, TextureSize.Y, 1),
        FIntVector(8, 8, 1));

    // 5. 注册 Compute Pass — 一行搞定！
    FComputeShaderUtils::AddPass(
        GraphBuilder,
        RDG_EVENT_NAME("MyCustomComputePass"),
        ComputeShader,
        PassParameters,
        GroupCount);
}

Compute Pass 优势

  • 无需 PSO 设置（无 Rasterizer/Blend/DepthStencil 状态）
  • 无需 Shader 绑定（无 VS+PS 组合）
  • 直接通过 UAV 读写，线程组内可共享 LDS (groupshared) 内存
  • FComputeShaderUtils::AddPass 封装所有样板代码
  • 天然支持 AsyncCompute，与图形管线并行

适用场景选择

  • 逐像素计算/滤波 → Compute Pass（更好的 cache 利用）
  • 需要硬件混合/插值/光栅化 → Raster Pass
  • 全屏后处理效果 → 两者皆可，Compute 通常更快
  • 复杂数据结构的并行处理 → Compute Pass（LDS 可用）
  • Simple pixel shader + depth → Raster Pass（利用 HiZ Early-Z）

5. 资源绑定详解

5.1 RDG 资源创建

API用途示例
GraphBuilder.CreateTexture(Desc, Name) 创建 Transient RDG 纹理。只在当前帧有效，不占用持久显存 CreateTexture(FRDGTextureDesc::Create2D(Extent, PF_FloatRGBA, FClearValueBinding::Black, TexCreate_ShaderResource | TexCreate_UAV), TEXT("MyTexture"))
GraphBuilder.CreateBuffer(Desc, Name) 创建 Transient RDG Buffer CreateBuffer(FRDGBufferDesc::CreateStructuredDesc(4, Count), TEXT("MyBuffer"))
GraphBuilder.RegisterExternalTexture(PooledRT) 将持久资源包装为 RDG 引用，使其可被 Pass 引用 RegisterExternalTexture(SceneTextures.Color.Target)
GraphBuilder.CreateUAV(Texture, [Mip]) 创建纹理的 UAV 视图（用于 Compute Pass 写入） CreateUAV(OutputTexture)
GraphBuilder.CreateSRV(Texture) 创建纹理的 SRV 视图（显式声明时使用） CreateSRV(FRDGTextureSRVDesc::Create(InputTexture))
GraphBuilder.AllocParameters<T>() 异步分配 Pass 参数结构体内存 AllocParameters<FMyPassCS::FParameters>()

5.2 资源状态自动转换

RDG 的最大优势是 自动资源屏障。当你在不同 Pass 中使用同一个 RDG 纹理时：

  1. Pass A 声明纹理为 RenderTarget → RDG 标记为 RENDER_TARGET 状态
  2. Pass B 声明同一纹理为 SRV 输入 → RDG 在 Pass A 和 B 之间自动插入 RENDER_TARGET → PIXEL_SHADER_RESOURCE 转换
  3. Pass C 声明同一纹理为 UAV → RDG 再插入 PIXEL_SHADER_RESOURCE → UNORDERED_ACCESS 转换

注意：UE5.8 中 SHADER_PARAMETER_RDG_TEXTURE 宏自动推导为 SRV 输入，SHADER_PARAMETER_RDG_TEXTURE_UAV 为 UAV。如果同时需要 SRV 和 UAV（例如对同一纹理做读写），必须分开声明两个参数，指向同一个 RDG 纹理。

6. Pass 间数据传递

6.1 RDG 纹理传递（最常见）

// ---- 上游 Pass：产生纹理 ----
FRDGTextureRef IntermediateTexture = GraphBuilder.CreateTexture(...);
AddMyCustomPass(GraphBuilder, View, SceneColor, /* Out */ IntermediateTexture);

// ---- 下游 Pass：消费纹理 ----
AddMyDownstreamPass(GraphBuilder, View, IntermediateTexture);

// RDG 自动分析：IntermediateTexture 在 MyCustomPass 中是
// Texture_W (来自 PassParameters 绑定信息), 在 MyDownstreamPass 中是 Texture_R，
// 自动插入 Resource_Write → Resource_Read 屏障

6.2 Buffer 数据回读

// 创建可回读 Buffer
FRDGBufferDesc BufferDesc = FRDGBufferDesc::CreateBufferDesc(sizeof(FVector4f), NumElements);
BufferDesc.Usage |= BUF_SourceCopy; // 必须标记 BUF_SourceCopy
FRDGBufferRef GPUBuffer = GraphBuilder.CreateBuffer(BufferDesc, TEXT("GPUBuffer"));

// 在下游 Pass 中执行回读
AddReadbackBufferPass(GraphBuilder, RDG_EVENT_NAME("Readback"), GPUBuffer,
    [](FRDGAsyncTask, FRHICommandList& RHICmdList)
    {
        // 此回调在 GPU 完成后触发，此时可以安全读取 Host 端内存
    });

6.3 RDG 黑板 — 跨 Pass 共享数据

// 使用 GraphBuilder.Blackboard 在互不相识的 Pass 间传递数据
struct FMyPassData
{
    FRDGTextureRef ResultTexture = nullptr;
    int32 ResultValue = 0;
};

// 上游 Pass 写入黑板
auto& BlackboardData = GraphBuilder.Blackboard.Create<FMyPassData>();
BlackboardData.ResultTexture = MyOutputTexture;
BlackboardData.ResultValue = 42;

// 下游 Pass（可能在完全不同的文件中）读取黑板
const auto& BlackboardData = GraphBuilder.Blackboard.Get<FMyPassData>();
FRDGTextureRef InputFromOtherPass = BlackboardData.ResultTexture;

7. 端到端：从零添加一个自定义 Pass

以下是完整步骤清单，假设我们要添加一个"将场景颜色乘系数后输出"的 Compute Pass：

步骤操作文件关键内容
1 编写 Shader Engine/Shaders/Private/
MyCustomPass.usf		 定义 MainCS 入口函数，声明纹理参数，[numthreads] 分组
2 声明 C++ Shader 类 Renderer/Private/
MyCustomPassRendering.h		 class FMyCustomPassCS : FGlobalShader + DECLARE_GLOBAL_SHADER + SHADER_PARAMETER_STRUCT
3 IMPLEMENT 注册 Renderer/Private/
MyCustomPassRendering.cpp		 IMPLEMENT_GLOBAL_SHADER(..., SF_Compute) + AddMyCustomPass() 函数实现
4 声明函数接口 Renderer/Private/
MyCustomPassRendering.h		 void AddMyCustomPass(FRDGBuilder&, const FViewInfo&, FRDGTextureRef SceneColor)
5 在 Render 函数中调用 Renderer/Private/
DeferredShadingRenderer.cpp		 在选定插入点（如 BasePass 之后）调用 AddMyCustomPass(GraphBuilder, View, SceneTextures.Color.Target)
6 添加 Build.cs 依赖 Renderer/Renderer.Build.cs 确认已有 "RenderCore" 依赖（通常不需要额外添加）
7 编译验证 — 编译引擎 → 启动编辑器 → 确认 Pass 在 GPU Profiler 中可见

调试技巧

  • GPU Profiler: 在编辑器中使用 stat gpu 或 profilegpu，自定义 Pass 名称会出现在时间线上
  • RenderDoc: 捕获一帧，搜索你的 Pass 名称，检查输入/输出纹理是否正确
  • Shader 编译错误: 检查 Output Log → 搜索 Shader 文件名，定位编译错误行号
  • 断言/崩溃: RDG 会验证参数完整性——如果某个 SHADER_PARAMETER 未赋值，RDG 在执行前就会 assert 而不是崩溃在 GPU 端

8. CVar 控制 Pass 开关和参数

通过 TAutoConsoleVariable 让自定义 Pass 支持运行时开关和参数调节：

#include "HAL/IConsoleManager.h"

// 0 = 禁用, 1 = 启用
static TAutoConsoleVariable<int32> CVarMyCustomPass(
    TEXT("r.MyCustomPass.Enable"),
    0,
    TEXT("0: Disable my custom pass\n1: Enable my custom pass"),
    ECVF_RenderThreadSafe);

// 强度参数
static TAutoConsoleVariable<float> CVarMyCustomPassIntensity(
    TEXT("r.MyCustomPass.Intensity"),
    1.0f,
    TEXT("Intensity multiplier for my custom pass"),
    ECVF_RenderThreadSafe);

// 在 Pass 调用处使用 CVar
void AddMyCustomPassIfEnabled(FRDGBuilder& GraphBuilder, const FViewInfo& View, FRDGTextureRef SceneColor)
{
    if (CVarMyCustomPass.GetValueOnRenderThread() == 0)
        return;  // 禁用时不注册任何 Pass

    float Intensity = CVarMyCustomPassIntensity.GetValueOnRenderThread();

    // ... 创建 Pass，使用 Intensity ...
}

CVar 速查表

CVar类型说明
ECVF_RenderThreadSafe—可在渲染线程读取，大多数渲染 CVar 都设此标志
ECVF_Cheat—仅开发版本可用，Shipping 版本不暴露
ECVF_ReadOnly—不可运行时修改
ECVF_Scalability—受 Scalability 系统管理，会根据画质档位自动调整
注意：CVar 读取函数必须与标志匹配。如果 CVar 标记了 ECVF_RenderThreadSafe，必须在 Render Thread 上调用 GetValueOnRenderThread()。在 Game Thread 上读 RenderThreadSafe 标志的 CVar 会导致未定义行为。

9. 常见问题与排错

问题原因解决
Pass 不执行 RDG 裁剪了"无消费者"的资源链路 确保 Pass 的输出资源被下游 Pass 引用，或以 GraphBuilder.AddDispatchHint() 强制保留
Shader 编译失败 .usf 文件路径不正确或 #include 缺失 检查 IMPLEMENT_GLOBAL_SHADER 中的虚拟路径（/Engine/Private/xxx.usf）与文件系统路径一致
断言：Resource not found SHADER_PARAMETER 声明的参数未赋值 RDG 在 Pass 注册时验证参数结构体所有字段是否已赋值。检查是否遗漏了某个参数的赋值
Compute Pass 输出全黑 线程组覆盖不全或 UAV 未创建 检查 GroupCount 计算是否覆盖全部输出区域，检查是否调用了 CreateUAV
GPU 崩溃/UAV 竞争 ReadWrite UAV 对同一位置有竞争写入 使用原子操作（InterlockedAdd/InterlockedMax），或使用单独的输出 Buffer
Pass 名称在 GPU Profiler 中不可见 Pass 被 RDG 合并或裁剪 使用 RDG_EVENT_NAME 宏给 Pass 命名，确认它没有被 Cull 掉
