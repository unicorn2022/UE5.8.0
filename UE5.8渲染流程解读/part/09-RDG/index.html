1. 管线中的位置

RDG（Render Dependency Graph）是 UE5.8 渲染管线的核心调度框架，FRDGBuilder 负责管理所有渲染 Pass 的依赖关系、资源生命周期和自动屏障插入，是整个延迟渲染流程的"骨架"。

2. 核心概念

RDG 的核心概念包括 Pass、资源（Texture/Buffer）、依赖边、Transient 资源分配和自动屏障系统，这些共同构成了 GPU 管线的高效调度模型。

3. Pass 注册与执行

每个渲染 Pass 通过 AddPass 注册到 FRDGBuilder 中，声明其输入/输出资源，RDG 自动分析跨 Pass 依赖关系并决定执行顺序。

4. 资源生命周期

RDG 资源由 FRDGBuilder 统一管理生命周期，从创建到最后一次被引用结束后自动释放，避免手动资源管理的内存泄漏和使用后释放（UAF）问题。

5. Transient 资源分配

TransientResourceAllocator 负责临时资源的池化分配与别名复用，通过分析资源活跃区间（Live Range）将不重叠的资源分配到同一物理显存块，显著降低显存占用。

6. 依赖分析与自动屏障

RDG 根据 Pass 间资源的读写关系自动插入 RHI 屏障（UAV Barrier、Render Target Transition 等），确保正确的 GPU 数据依赖和缓存一致性。

7. 自定义 RDG Pass 完整模板

从 Shader 参数结构体定义到 AddPass 注册，展示一个完整的自定义 RDG Pass 编写模板，包含 Shader 实现和 C++ 调度代码。

8. RDG 验证层

RDG 内置验证层可以检测未使用的资源声明、资源访问冲突、Pass 执行顺序错误等问题，是开发阶段定位渲染 bug 的重要工具。

9. RDG Dump 与调试

通过 GraphViz 导出、调试标记和 RDG Insights 等工具，可视化分析实际的渲染图依赖结构，定位性能瓶颈和依赖错误。

10. 性能与陷阱

RDG 使用中的常见性能陷阱，包括过度细分的 Pass 导致编译开销过大、不必要的资源拷贝、Transient 资源过度分配等问题及最佳实践。

11. CVar 速查

与 RDG 相关的常用 Console Variable 参考，涵盖调试、验证、Dump 和性能调优相关的命令行开关。
